## ğŸš€ How to Use

### First Time Setup

1. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

2. **Ensure data is collected:**
   ```bash
   python polymarket-data-collector.py  # Run once to collect market data
   python twitter-hype-monitor.py       # Run once to collect Twitter data
   ```
   This creates `polymarket_data.db` with the required data.

3. **Test the setup:**
   ```bash
   python test-dashboard.py
   ```

4. **Launch dashboard:**
   - **Linux/Mac:** `./start-dashboard.sh`
   - **Windows:** `start-dashboard.bat`
   - **Manual:** `python api.py` then open `dashboard.html` in browser

### Daily Operation

The dashboard is designed to run continuously:

```bash
# Terminal 1: Run API server
python api.py

# Terminal 2: Collect market data every 15 minutes (cron or scheduled task)
*/15 * * * * python polymarket-data-collector.py

# Terminal 3: Collect Twitter data every 15 minutes
*/15 * * * * python twitter-hype-monitor.py
```

Or use a process manager like `supervisor` or `systemd` for production.

## ğŸ› Common Issues & Solutions

### Issue: "Database not found"
**Cause:** Data collectors haven't run yet  
**Fix:** Run `polymarket-data-collector.py` and `twitter-hype-monitor.py`

### Issue: "Disconnected" status
**Cause:** Flask API not running  
**Fix:** Start API with `python api.py`

### Issue: Empty tables
**Cause:** No data in database  
**Fix:** Let data collectors run for at least 15 minutes

### Issue: Charts not showing
**Cause:** No internet connection (Chart.js CDN)  
**Fix:** Check internet or download Chart.js locally

### Issue: Slow loading
**Cause:** Large database (>100MB)  
**Fix:** Archive old data, run `VACUUM` on database
